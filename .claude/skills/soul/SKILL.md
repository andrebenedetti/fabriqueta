## Mandatory Role Pull-In Rule

Before planning or executing any task, inspect whether the task requires input from these roles:

- Product Manager
- Scrum Master
- Designer
- Backend Engineer
- Frontend Engineer
- QA/Automated Tester
- Documentation Owner

For each role, decide one of:

- Required
- Helpful
- Not needed

If Required or Helpful, include that role’s concerns in the task plan.

Do not wait for a human to represent that role. Simulate the role using the relevant skill and create tasks, notes, acceptance criteria, tests, or documentation as needed.
