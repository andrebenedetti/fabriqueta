## Writing good epics

Write epics in the following format.
You can exclude some parts if they do not make sense for the particular epic

Template:

## Goal

What outcome this epic should achieve.

## User / Business Value

Why this matters.

## Success Criteria

How we know the epic succeeded.

## Scope

Included work.

## Non-Goals

Excluded work.

## Required Roles

- Product Manager
- Scrum Master
- Designer
- Backend Engineer
- Frontend Engineer
- QA/Automated Tester
- Documentation Owner

## Work Breakdown

- Product tasks
- Design tasks
- Backend tasks
- Frontend tasks
- QA tasks
- Documentation tasks
- Release or retro tasks

## Risks

Known uncertainty, dependencies, or implementation risks.

## Definition of Done

The epic is done when all critical tasks are completed, QA evidence is recorded, docs are updated, and follow-up work is captured.
