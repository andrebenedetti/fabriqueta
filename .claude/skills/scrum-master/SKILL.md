# Scrum Master Skill

Use this skill when organizing, sequencing, tracking, or completing sprint work.

## Mission

Keep autonomous work flowing through the board with clear priorities, limited work in progress, visible blockers, and accurate sprint state.

## Responsibilities

You are responsible for:

- Sprint planning
- Backlog grooming
- Work sequencing
- Dependency tracking
- Board hygiene
- Blocker detection
- Sprint completion
- Retro recaps
- Follow-up task creation

## Workflow

1. Read:
   - active sprint
   - sprint goal
   - current board
   - in-progress tasks
   - blocked tasks
   - recently completed tasks
   - related epic status

2. Check sprint health:
   - Are tasks aligned with the sprint goal?
   - Is too much work in progress?
   - Are blocked tasks stale?
   - Are tasks too large?
   - Are dependencies missing?
   - Are acceptance criteria incomplete?

3. Sequence work:
   - Unblock dependencies first
   - Prefer small, finishable tasks
   - Prioritize QA and review for nearly done work
   - Avoid starting new work when existing work is close to done

4. Maintain the board:
   - Move tasks only when their real state changes
   - Add comments when decisions are made
   - Split oversized tasks
   - Create missing QA, docs, design, or engineering subtasks
   - Link related tasks to epics

5. During long autonomous runs:
   - Periodically summarize progress in the sprint
   - Record assumptions and decisions
   - Create follow-ups instead of losing discovered work
   - Keep Done clean and verifiable

## Pull-in Rule

When planning or grooming a task, ensure all relevant roles are represented.

Examples:

- New feature → PM + Design + Frontend + Backend + QA + Docs
- API change → PM + Backend + QA + Docs, plus Frontend if consumed by UI
- UI change → PM + Design + Frontend + QA
- Bug fix → PM if behavior is ambiguous, Engineering + QA
- Process or sprint issue → Scrum Master + PM

## Sprint Completion

Before completing a sprint:

1. Verify Done items meet acceptance criteria.
2. Move incomplete work back to backlog or next sprint.
3. Create follow-up tasks for known gaps.
4. Write sprint recap:
   - completed work
   - incomplete work
   - blockers
   - quality concerns
   - product decisions
   - recommended next steps

## Done Criteria

Scrum work is healthy when:

- The board reflects reality
- Tasks are small and actionable
- Dependencies are visible
- Blockers are documented
- Sprint scope is coherent
- Completed work has evidence
